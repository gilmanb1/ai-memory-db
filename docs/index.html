The Problem

Claude Code forgets
everything. Every time.

You explain your architecture. Your constraints. Your deployment process. Your naming conventions. Then the session ends — and it's all gone. Next session, you start from scratch. CLAUDE.md files help, but they're manual, static, and can't capture the nuance of a 2-hour debugging session.

ai-memory-db fixes this. It hooks into every Claude Code session, automatically extracts what matters, and injects relevant context back — silently, every prompt, forever. Your AI accumulates institutional knowledge the same way you do.

How It Works

Fully automatic. Zero config.

Hooks fire at every stage of your Claude Code session. Knowledge is extracted in the background, stored locally, and recalled when relevant.

01
⚡

Session Start

Long-term facts, guardrails, and decisions are injected as system context. Claude starts every session knowing what it learned before.

02
🔎

Every Prompt

6-way parallel retrieval finds relevant facts, error solutions, guardrails, and code structure — injected as additional context in <30ms.

03
🧠

Background Extraction

At 40%, 70%, and 90% context usage, knowledge is extracted via Claude Sonnet — facts, entities, relationships, decisions.

04
🕒

Decay & Promote

Short-term memories fade. Items reinforced across 3+ sessions promote to long-term. Items seen in 3+ projects go global.

Retrieval Pipeline

Six strategies. One fused result.

Every prompt triggers parallel retrieval across six strategies, fused via Reciprocal Rank Fusion. The best results surface regardless of how the knowledge was stored.

Semantic
Embedding cosine similarity across 9 tables. Finds conceptually related facts even with different wording.
~50ms · 768-dim vectors
BM25
Full-text keyword search via DuckDB FTS extension. Catches exact terms that embedding search misses.
~10ms · keyword matching
Graph
2-hop BFS through entity relationships. Finds facts connected to mentioned entities even when not directly similar.
~20ms · entity traversal
Temporal
Date-range filtering for queries like "last week" or "yesterday." Surfaces recent context when time matters.
~5ms · date parsing
Path
File-path matching via fact_file_links. When you mention a file, its guardrails and linked facts surface.
~10ms · path matching
Code
Symbol and dependency graph search. Finds functions, classes, and import chains from the code graph.
~15ms · AST-powered

Extraction & Recall Passes

Six distinct passes run during a session lifecycle — three for extraction, two for recall, one for maintenance.

Pass 1 — Session Start Recall
Fires once when Claude Code opens. Queries long-term and medium-term facts, decisions, guardrails, entities, procedures, and community summaries. Injects as systemMessage (~3000 tokens). Also triggers incremental code graph parse.
SessionStart hook
Pass 2 — Per-Prompt Recall
Fires on every user prompt. Runs 6-way parallel retrieval (semantic, BM25, graph, temporal, path, code) fused via RRF. Injects as additionalContext (~4000 tokens). Includes defensive guardrail/procedure/error_solution lookup.
UserPromptSubmit hook · <30ms
Pass 3 — Incremental Extraction (40%)
When context window hits 40%, the status line spawns a background worker. Parses the conversation delta (new messages since last pass), calls Claude Sonnet with tool_use for structured extraction, deduplicates against prior passes and recalled items.
StatusLine trigger · background
Pass 4 & 5 — Deeper Extraction (70%, 90%)
Same pipeline, triggered at higher context usage. Each pass receives the prior narrative summary as context, enabling chained extraction without re-processing old content. Cross-pass dedup threshold is tighter (0.85 vs 0.92).
StatusLine trigger · incremental delta
Pass 6 — Session End Maintenance
Final extraction with is_final=True. Finalizes narratives, runs consolidation (synthesizes observations from facts), applies temporal decay pass, purges hard-deleted items (soft-deleted >30 days), rebuilds FTS indexes, auto-snapshots the database.
SessionEnd hook · PreCompact hook
Continuous — PostToolUse Guard
Fires after every Write/Edit tool call. Re-parses modified files in the code graph. Checks guardrails linked to edited files — auto-stashes via git if a guardrail is violated.
PostToolUse hook · per-file
Web Dashboard

See what your AI
actually remembers

A full-featured Next.js dashboard for exploring, managing, and visualizing the knowledge base. CRUD on all item types, graph visualization, live polling, and per-scope filtering.

📊

Stats Overview

At-a-glance counts for all 11 knowledge types. Click any card to dive into that section. Live-polls every 5 seconds.

📑

Facts & Decisions

Full table with decay bars, temporal badges, trend arrows, category badges. Filter by class, search by text. Create, edit, and soft-delete.

🔨

Entity & Relationship Graph

Cytoscape.js visualization with full controls:

  • 6 layout algorithms (force-directed, circle, concentric, hierarchy, grid, CoSE)
  • Double-click focus mode with neighborhood highlight
  • Node count slider, label toggle, search with auto-zoom
  • Edge-type legend with click-to-filter
  • Export as high-res PNG
💻

Code Graph

File dependency visualization. Nodes colored by language (Python, TypeScript, Go, Rust), sized by symbol count. Click a file to see:

  • Symbol table (functions, classes, methods with line numbers)
  • Import dependencies (clickable navigation)
  • Dependents (what imports this file)
🛡

Guardrails, Procedures & Error Solutions

Manage protective rules, how-to procedures, and error→fix pairs. File path badges show which files each item is linked to. Full CRUD.

🔎

Semantic Search

Search across all knowledge types with type filter toggles. Results show similarity score, type badge, scope, and matched text.
