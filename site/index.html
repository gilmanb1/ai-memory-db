How It Works

Fully automatic. Zero config.

Hooks fire at every stage of your Claude Code session. Knowledge is extracted in the background, stored locally, and recalled when relevant.

01
⚡

Session Start

Long-term facts, guardrails, and decisions are injected as system context. Claude starts every session knowing what it learned before.

02
🔎

Every Prompt

6-way parallel retrieval finds relevant facts, error solutions, guardrails, and code structure — injected as additional context in <30ms.

03
🧠

Background Extraction

At 40%, 70%, and 90% context usage, knowledge is extracted via Claude Sonnet — facts, entities, relationships, decisions.

04
🕒

Decay & Promote

Short-term memories fade. Items reinforced across 3+ sessions promote to long-term. Items seen in 3+ projects go global.

Features

Everything you need.
Nothing you don't.

📚

8 Knowledge Types

Facts, decisions, guardrails, procedures, error solutions, observations, entities, and relationships — each with distinct recall priority.

🌐

Project Scoping

Knowledge is isolated per git repo. Facts from project A never leak into project B. Cross-project items auto-promote to global.

🛠

22 Slash Commands

/remember, /forget, /reflect, /search-memory, /knowledge, /audit-memory, /export-memory, and 15 more for full control.

🤖

Code Graph

Multi-language AST parsing (Python, TypeScript, Go, Rust) via tree-sitter. Symbols, imports, and dependencies tracked per file.

🛡

Guardrail Enforcement

PostToolUse hook detects edits to guardrailed files and auto-stashes via git. Correction detection auto-supersedes bad facts.

📊

Web Dashboard

Next.js + Cytoscape.js dashboard with full CRUD, graph visualization, live polling, code graph, and per-scope filtering.

💾

Auto-Snapshots

DB snapshot on every session end (rotating last 5). Export to JSON, import from JSON, restore from any snapshot.

🔒

Local-First

Everything runs locally. DuckDB for storage, Ollama/ONNX for embeddings. Your knowledge never leaves your machine.

⚡

Graceful Degradation

Ollama down? Falls back to ONNX. API fails? Recall still works. DB locked? Retries with backoff. No crash, ever.

Retrieval Pipeline

Six strategies. One fused result.

Every prompt triggers parallel retrieval across six strategies, fused via Reciprocal Rank Fusion. The best results surface regardless of how the knowledge was stored.

Semantic
Embedding cosine similarity across 9 tables. Finds conceptually related facts even with different wording.
~50ms · 768-dim vectors
BM25
Full-text keyword search via DuckDB FTS extension. Catches exact terms that embedding search misses.
~10ms · keyword matching
Graph
2-hop BFS through entity relationships. Finds facts connected to mentioned entities even when not directly similar.
~20ms · entity traversal
Temporal
Date-range filtering for queries like "last week" or "yesterday." Surfaces recent context when time matters.
~5ms · date parsing
Path
File-path matching via fact_file_links. When you mention a file, its guardrails and linked facts surface.
~10ms · path matching
Code
Symbol and dependency graph search. Finds functions, classes, and import chains from the code graph.
~15ms · AST-powered
Get Started

Install in 60 seconds

One command. Restart Claude Code. That's it. The system runs automatically from that point forward — no configuration, no manual steps.

The installer copies the memory package, hook scripts, and slash commands to ~/.claude/. It configures hooks in your settings.json and runs the test suite to verify everything works.
